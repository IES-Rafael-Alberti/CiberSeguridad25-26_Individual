# Captura y Análisis de Tráfico de Red en Conexiones HTTP

## Objetivo de la Actividad

El objetivo de esta actividad es realizar una práctica de captura y análisis de tráfico de red centrada en las conexiones HTTP. Se busca comprender cómo se transmiten los datos a través de formularios web y reflexionar sobre los riesgos de seguridad asociados a este tipo de conexiones no cifradas.

---

## 1. Preparación de la Herramienta

### Instalación y configuración de Wireshark

Se descargó e instaló Wireshark desde su página oficial ([https://www.wireshark.org/](https://www.wireshark.org/)).
Durante la instalación se mantuvieron las opciones predeterminadas.
Una vez instalado, se abrió el programa con permisos de administrador para asegurar el acceso a todas las interfaces de red disponibles.

### Familiarización con la interfaz

En la parte superior de la ventana se encuentran los menús principales (File, Capture, Analyze, etc.).
En la barra de filtros se puede escribir el tipo de tráfico que se desea capturar.
La interfaz principal se divide en tres secciones:

1. Lista de paquetes capturados.
2. Detalles del paquete seleccionado.
3. Visualización en formato hexadecimal.

---

## 2. Captura de Tráfico HTTP

Se seleccionó la interfaz de red correspondiente a la conexión activa (en este caso, Wi-Fi).
Para limitar la captura únicamente al tráfico HTTP, se escribió el siguiente filtro en la barra superior:

```
http
```

Posteriormente se inició la captura presionando el botón con el ícono de la aleta de tiburón.

---

## 3. Conexión a un Sitio Web

Con Wireshark ejecutándose, se abrió un navegador web y se accedió al siguiente sitio:

```
http://testphp.vulnweb.com/login.php
```

En el formulario de inicio de sesión se introdujeron los siguientes datos de prueba:

```
Usuario: testuser
Contraseña: 123456
```

Luego se presionó el botón “Login” para enviar el formulario.
Una vez enviada la solicitud, se detuvo la captura en Wireshark con el botón de detener (cuadro rojo).

---

## 4. Análisis del Tráfico Capturado

Para enfocar el análisis en las solicitudes y respuestas HTTP, se aplicó el filtro:

```
http.request || http.response
```

En los resultados se observó una petición **HTTP POST** hacia el archivo `login.php`.
Dentro de la sección “Hypertext Transfer Protocol” se pudieron visualizar los datos transmitidos:

```
POST /login.php HTTP/1.1
Host: testphp.vulnweb.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 29

uname=testuser&pass=123456
```

Esto demuestra que los datos del formulario (usuario y contraseña) fueron enviados en texto plano, sin ningún tipo de cifrado.

## 5. Observaciones y Conclusiones

Durante la práctica se comprobó que las conexiones HTTP no ofrecen seguridad en la transmisión de datos.
Las credenciales y la información introducida en formularios pueden ser vistas fácilmente desde herramientas como Wireshark.

### Observaciones sobre la seguridad

* El protocolo HTTP transmite la información sin cifrar.
* Cualquier usuario conectado a la misma red puede interceptar y leer los datos.
* HTTPS, en cambio, utiliza cifrado TLS/SSL, lo que impide que la información sea legible por terceros.

### Medidas de mejora

1. Utilizar HTTPS en todos los formularios que manejen datos sensibles.
2. Implementar autenticación segura (por ejemplo, tokens o encriptación de contraseñas).
3. Evitar el uso de HTTP en páginas de inicio de sesión o formularios de registro.
4. Mantener certificados SSL actualizados y válidos.

---

## 6. Conclusión General

La práctica permitió comprender cómo funciona la transmisión de datos en conexiones HTTP y los riesgos asociados a su uso.
Se observó de forma práctica cómo la información de un formulario puede ser interceptada en texto plano, lo que demuestra la importancia de utilizar protocolos seguros como HTTPS para proteger la privacidad y la integridad de los datos transmitidos en la web.
