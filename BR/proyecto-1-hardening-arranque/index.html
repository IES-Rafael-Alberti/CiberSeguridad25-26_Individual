El presente Codelab documenta las guías técnicas para el bastionado (hardening) de la secuencia de arranque de un sistema informático, estructurado en dos fases:

  1. Hardening de BIOS/UEFI (Parte 1): Configuración del firmware para mitigar el acceso no autorizado y la manipulación del orden de inicio.

El establecimiento de contraseñas en el firmware es una medida esencial para impedir la modificación de la configuración de seguridad y evitar el arranque del sistema por parte de personal no autorizado.

Contraseña de Administración o Supervisor

Esta clave es crítica, ya que su conocimiento es necesario para acceder y modificar la totalidad de los parámetros de la BIOS/UEFI. Su omisión permite a cualquier usuario deshabilitar las protecciones implementadas.

  1. Acceso al Firmware: Reinicie el equipo y pulse la tecla de acceso al Setup (usualmente F2, Supr, F10, o F12) durante la fase inicial de arranque.
  2. Ubicación: Navegue a la sección Security (Seguridad), Password o System Setup.
  3. Configuración: Defina la Administrator Password (o Supervisor Password).
  4. Recomendación: Se recomienda utilizar una clave robusta y única, distinta de las credenciales de usuario del sistema operativo.

Contraseña de Usuario o Arranque (Power-On Password - POP)

Esta contraseña impide la carga del sistema operativo y es solicitada antes de la inicialización de cualquier componente del sistema operativo (SO).

  1. Ubicación: Localice la opción dentro de la misma sección de Seguridad.
  2. Configuración: Habilite y defina la User Password o Power-On Password (POP).
  3. Aplicación: Verifique que la opción Password Check o Password on boot esté activada para garantizar la solicitud de la clave en cada inicio.

Orden de Arranque Seguro

La definición estricta de la secuencia de dispositivos de inicio previene que un actor malicioso utilice soportes extraíbles (USB o DVD) para cargar un sistema operativo alternativo o herramientas de ataque.

  1. Navegación: Acceda a la pestaña Boot (Arranque) o Boot Priority.
  2. Prioridad: Establezca el disco principal (HDD/SSD con el SO) como Boot Option #1 (máxima prioridad).
  3. Secundarios: Relegue el resto de opciones de arranque (USB, CD/DVD, Red) a posiciones inferiores.

Deshabilitar Arranques desde Dispositivos Externos

A pesar de controlar la prioridad, es una práctica de endurecimiento recomendada deshabilitar por completo el soporte para el inicio desde dispositivos extraíbles si no son necesarios para el funcionamiento habitual.

  • Ubicación: Generalmente en las pestañas Boot o Advanced.
  • Acciones: Deshabilite explícitamente las siguientes fuentes:
    • Boot from USB / USB Mass Storage: Establecer como Disabled.
    • PXE Boot / Network Boot: Establecer como Disabled (salvo que se requiera para gestión remota).
    • Boot Menu Key: Si la opción está disponible, deshabilite la tecla de acceso directo al menú de arranque (ej. F12) para forzar la interacción con la BIOS protegida.

Secure Boot y Módulo de Plataforma Confiable (TPM)

Estas características representan medidas de seguridad avanzadas a nivel de hardware y firmware introducidas con la especificación UEFI.

A. Secure Boot

Asegura que el firmware solo cargue cargadores de arranque (boot loaders) y drivers que hayan sido firmados digitalmente por autoridades de confianza.

  1. Requisito: El firmware debe operar en modo UEFI.
  2. Ubicación: Sección Security o Boot.
  3. Acción: Verifique que Secure Boot esté configurado como Enabled.

B. Trusted Platform Module (TPM)

Es un coprocesador criptográfico diseñado para almacenar claves y realizar mediciones de integridad del proceso de arranque. Es fundamental para tecnologías de cifrado de disco completo.

  1. Ubicación: Sección Security o Advanced.
  2. Acción: Localice el TPM Device (o la implementación específica del fabricante como Intel PTT o AMD fTPM) y establézcalo en Enabled.
